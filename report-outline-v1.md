## 评测报告大纲（11 章节）

```
# {产品名} 评测报告

## 0. 摘要 / Executive Summary
  - 一句话定位 + 一句话结论
  - 适用人群 + 推荐使用方式
  - （引用来源：综合 A/E）

## 1. 产品概述与背景
  - 产品是什么、解决什么问题
  - 公司背景、发布时间、定价、目标用户
  - CEO/官方公开声明与承诺
  - （引用来源：E 公司声明、H 先前报道）

## 2. 测试环境与方法
  - 测试场景、设备、用例集
  - 评测维度与判定标准
  - （引用来源：A 作者亲测——方法论透明度）

## 3. 作者亲测实录
  - 实际使用流程 walkthrough
  - 截图/照片证据
  - 第一手体感（速度、准确度、易用性）
  - （引用来源：A 作者亲测）

## 4. 核心能力实测
  - 按场景分小节：Q&A / 创作 / 推理 / 工具调用 等
  - 每个场景给：测试用例 → 结果 → 评价
  - 成功案例 + 失败案例并列
  - （引用来源：A 作者亲测、C 学术研究——如相关）

## 5. 竞品对比
  - 与 1-2 个主要竞品并排对比
  - 功能矩阵 / 体验差异 / 适用场景区分
  - （引用来源：D 竞品对比、A 作者亲测）

## 6. 专家与学术视角
  - 命名专家访谈（姓名+机构+引语）
  - 相关学术研究引用（期刊/预印本 + 方法描述）
  - 专家社交媒体反应（如争议性话题）
  - （引用来源：B 专家访谈、C 学术研究、J 社交媒体）

## 7. 公司承诺 vs 现实落差
  - 对照官方声明/roadmap 与实测结果
  - 已兑现 / 未兑现 / 待观察
  - （引用来源：E 公司声明、A 作者亲测）

## 8. 真实用户场景与案例
  - 用户故事/截图
  - 研究者观察到的真实使用案例
  - 边界 case 与失败模式
  - （引用来源：G 真实案例、C 学术研究）

## 9. 数据与行业背景
  - 使用量数据、行业规模数据
  - 监管/政策背景
  - （引用来源：F 数据统计、I 官方文件、H 先前报道）

## 10. 局限性与风险
  - 功能缺失
  - 安全/隐私/伦理风险
  - 专家警示
  - （引用来源：B 专家访谈、E 公司声明、I 监管文件）

## 11. 评测结论与建议
  - 综合定位
  - 适用人群建议
  - 待观察项 / 后续跟踪
  - 推荐使用方式
  - （引用来源：综合 A/B/E）

## 附录：引用来源
  - 公司官方材料（E）
  - 专家访谈记录（B）
  - 学术研究链接（C）
  - 先前报道链接（H）
  - 数据来源（F）
```
