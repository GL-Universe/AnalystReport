## 一、观点转变

### 1.1 之前的立场
- 早期视频中强烈推荐 low/no reasoning 模式
- 认为 X-High 推理浪费 token,只适合数学/复杂逻辑

### 1.2 现在的认知
- X-High 推理释放了强大的自动化工作流
- 同样的功能在 low reasoning 下也能实现,但容易出错
- "用 X-High 一次性成功完成任务" > 用 low 多次迭代

---

## 二、Computer Use (电脑操控)

### 2.1 什么是 Computer Use
- GPT-5.5 能够直接控制用户桌面
- 查看屏幕、移动鼠标、点击、输入文字
- 可以测试和验证自己写的代码

### 2.2 自动化测试前端
- 写好代码 → 模型打开浏览器 → 自己查看效果 → 发现 bug → 修复 → 重复验证
- 无需人工介入每个测试步骤
- 极大的效率提升(一次性完成闭环)

### 2.3 风险讨论
- "我需要给我的 AI 打开所有危险权限"
- 完全信任模型操控桌面是安全风险
- 但为了让模型去真正"完成工作",完全的自主权是必要的

---

## 三、自动化与自主权

### 3.1 让 Agent 自主运行
- 可以做"不是快速能做完的任务"(如大型迁移、复杂重构)
- 允许模型运行数小时甚至过夜(无人监督)
- "一旦过了信任那道坎儿,效率是十倍的提升"

### 3.2 危险权限的意义
- 给模型 sudo 权限、文件系统全访问
- 能够在系统层面做变更、安装依赖、配置环境
- 这是从"copilot"到"autopilot"的关键转变

### 3.3 Codex Desktop 改进
- 桌面版 Codex 支持更多系统级操作
- 能在 VS Code 和其他编辑器间切换
- 移动版也已上线,可远程控制桌面代理

---

## 四、工作树(Worktrees)和并行化

### 4.1 工作树概念
- 在同一仓库创建多个并行分支
- 每个分支可以独立运行不同的 AI 任务
- 在 Pi 中通过特定命令操作

### 4.2 实际应用
- 允许同时处理多个特性/修复
- 每个工作树可以配置不同的推理级别
- 高并发开发成为可能

---

## 五、Peter's Experiment: 130万美元/月 API 支出

### 5.1 实验背景
- Peter 每月花费 $130 万美元在 API 调用上
- 主要用于 GPT-5.5 Pro 的 X-High 推理

### 5.2 发现
- X-High 推理在复杂任务上的成功率远高于其他级别
- 如果没有这个级别,任务将无法完成
- 这解释了为什么 Pro 版 $30/$180 的价格仍有人付费

---

## 六、Codex 移动应用

### 6.1 新功能
- 移动版 Codex 可以控制桌面代理
- 随时随地发送任务,让桌面模型继续运行
- 工作流可跨平台同步

### 6.2 使用场景
- 离开电脑后用手机检查进度
- 在自己不方便时发起新任务
- 24/7 的开发代理

---

## 七、关键结论

1. X-High 推理是解锁 GPT-5.5 全部潜力的必要模式
2. Computer Use 是未来,信任模型自主操控是必须的思维转变
3. 并行工作树让开发效率成倍提升
4. 真正高投入(每月百万美元级别)的场景验证了 X-High 推理的商业价值
5. Codex 移动应用让 AI 代理真正实现随时随地开发
