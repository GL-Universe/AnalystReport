## 一、模型性能(基准测试)

### 1.1 GDPval (84.9%)
- **用 44 种真实职业任务评估模型**(分析数据、写报告、做判断)
- 不是传统选择题,比 GPT-5.4 提升,高于 Opus 4.7 (80.3%) 和 Gemini 3.1 Pro (67.3%)

### 1.2 OSWorld (78.7%)
- 衡量模型在**真实电脑环境中的操作能力**
- 模型不仅能告诉你怎么做,还能直接替你去做(点击界面、切换工具、执行多步骤)

### 1.3 Tau2 Telecom (98.0%)
- 电信客服流程测试(无需额外调优)
- 贴近企业真实工作: 复杂、多步骤、上下文依赖

### 1.4 编程相关
- **Terminal-Bench 2.0**: 82.7%
- **SWE-Bench Pro**: 58.6%

### 1.5 知识工作相关
- **FinanceAgent**: 60.0%
- **内部投资银行建模**: 88.5%
- **OfficeQA Pro**: 54.1%

### 1.6 ARC-AGI-2
- 取得 **85.0%** 准确率,成为新 SOTA

## 二、评价标准的转变

### 2.1 从"知识问答"到"任务完成"
- 过去: MMLU, GPQA 看知识/推理
- 现在: GDPval, OSWorld 看任务级执行
- **关键转变: 从"知不知道"到"能不能做完"**

### 2.2 GPT-5.5 的任务处理模式
1. 获取信息 → 2. 做判断 → 3. 必要时调用工具 → 4. 整理成可用输出
- 在编程: 参与整个开发流程而不只是生成代码
- 在知识工作: 产出报告/模型/决策建议而不只是提供答案
- 在操作: 直接进入电脑环境执行
- **定位转变: 从"回答"转向"执行"**

---

## 三、效率与定价

### 3.1 Token 效率
- 速度与 GPT-5.4 基本持平
- 但完成同样 Codex 任务使用**明显更少 token**
- 对 API 用户直接决定真实成本

### 3.2 定价
- GPT-5.5: $5/$30 每百万 token
- Pro 版本更高
- 是 GPT-5.4 的 2 倍

### 3.3 OpenAI 的成本逻辑
- 单价提升,但因任务完成效率高,总成本未必上升

### 3.4 安全体系
- 当前防护最严格的一代
- 内部/外部红队测试 + 近 200 个真实场景调整

## 四、编码表现(Codex 中)

### 4.1 官方 Codex Demo
- **天体图片 → Web 应用** (WebGL 3D渲染 + Vite + NASA Artemis II 真实数据)
- **3D 地牢竞技场原型** (Three.js + 战斗系统 + 敌人机制 + UI反馈, 角色和动画第三方处理)

### 4.2 工程级表现
- 大任务中保持上下文(不只盯一段代码)
- 问题不明确时能推理故障原因
- 用工具验证假设
- 修改贯穿整个代码库

### 4.3 内部使用
- OpenAI 内部 **85% 员工每周使用 Codex**
- 覆盖软件工程、财务、传播、市场、数据科学、产品

## 五、基础能力实测

### 5.1 物理理解
- 鹈鹕骑自行车: GPT-5.5 vs GPT-5.4 对比改进明显
- 六边形小球滚动: 物理理解增强

### 5.2 审美设计
- 高端品牌网站(单 HTML 文件): 排版、间距、视觉一致性
- Unity 风格复杂 SVG 动画

### 5.3 生活常识
- 洗车问题: 需要"理解用户真实意图"而非刻板回答

## 六、模型演进脉络(GPT-4o → GPT-5.5)

### 6.1 GPT-4o 时代
- 文本+图像+语音放入同一个模型
- 多模态能力统一

### 6.2 GPT-5 时代
- 模型多了判断层: 多快响应、多深推理、是否调用工具

### 6.3 GPT-5.3
- 编码和工具调用明显强化
- 开始稳定完成多步骤代码生成/调试/执行

### 6.4 GPT-5.4
- 重点转向 computer use 和工作流
- 响应速度/Token 利用率/长任务稳定性持续优化

### 6.5 GPT-5.5 的位置
- 不只是性能提升,继续推进到任务执行
- 更像"后台持续运转的系统"而非一次性问答
- 用户关系演进: 一问一答 → 交一件事,看它一步步做完

---

## 七、关键结论

1. GPT-5.5 从"答题模型"进化为"执行模型",能自主组织步骤完成任务
2. 评价标准改变: 不看知不知道,看能不能做完
3. Token 效率大幅提升,2x定价实际成本增加有限
4. 代码/知识工作/电脑操作三方面均可直接投入生产
5. OpenAI 内部已高度依赖 Codex(85% 员工每周使用)
6. 安全同步升级,目前最严格的一代
