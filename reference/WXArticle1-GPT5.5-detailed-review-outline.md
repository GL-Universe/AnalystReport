## 二、编程能力深度分析

### 2.1 Terminal-Bench 2.0 (82.7%)
- 衡量命令行与终端环境下的多步骤排错与系统管理能力
- 显著领先 Claude Opus 4.7

### 2.2 SWE-Bench Pro (58.6%)
- 真实软件工程任务: 在大型代码库中定位 bug、跨文件修改
- 但 Claude Opus 4.7 在 SWE-Bench Pro 上的表现未被完全引用,暗示差距缩小

### 2.3 Expert-SWE (73.1%)
- 专业级软件工程基准
- 体现模型从"写代码"到"完成整个开发过程"的能力演化

### 2.4 Codex 中的优势
- 400K token 上下文窗口
- 能够在整个项目级别进行代码维护
- Codex 成为"超级应用"的战略核心

---

## 三、知识工作能力

### 3.1 GDPval (84.9%)
- 44 种真实职业任务评估,包括分析数据、写报告、做判断
- 相比 GPT-5.4 (83.0%) 提升
- 高于 Claude Opus 4.7 (80.3%) 和 Gemini 3.1 Pro (67.3%)

### 3.2 OSWorld (78.7%)
- 真实 OS 操作环境评估: 点击界面、切换工具、执行多步骤
- 实际演示了模型"动手做"而非仅仅"口头回答"的能力

### 3.3 内部使用数据
- OpenAI 内部 **85% 的员工每周使用 Codex**
- 覆盖软件工程、财务、传播、市场、数据科学、产品等多个团队

---

## 四、科学研究能力

### 4.1 GeneBench / BixBench
- 生物信息学与化学相关的科研任务
- 分数提升相对温和,但出现了"参与推理、验证甚至辅助发现新结果"的案例

### 4.2 Ramsey 数证明
- 参与了 Ramsey 理论中的一个新证明
- 展示了 AI 在纯数学研究中的辅助作用

### 4.3 研究转变
- 从"可回答问题"到"可辅助研究"的能力边界扩展

---

## 五、推理效率与底层优化

### 5.1 推理效率提升
- 完成同样任务使用 **更少 token**
- 20% 的速度改进: 通过 Codex 优化的推理分区技术
- 端到端延迟更低

### 5.2 GPU 协同设计
- 在英伟达 GB200/GB300 上协同开发
- 模型自身参与优化推理栈

### 5.3 Altman 的定位转变
- Sam Altman: OpenAI 正逐渐成为一家 **AI 推理公司**
- 推理成本是决定 AI 产品化的核心因素

---

## 六、安全与防护(Preparedness Framework)

### 6.1 高风险评定
- **Cybersecurity: High** (首次被评为 High)
- 生物/化学能力也是 High

### 6.2 安全评估流程
- 内部与外部红队测试
- 近 200 个真实使用场景调整
- 当前防护最严格的一代模型

### 6.3 Bio Bug Bounty 项目
- 启动生物安全漏洞悬赏
- 最高奖金 **$25,000**
- 邀请外部研究者帮助发现潜在风险

---

## 七、定价与可用性

### 7.1 API 定价
- GPT-5.5: $5 / $30 每百万 token (输入/输出)
- GPT-5.5 Pro: $30 / $180
- 价格是 GPT-5.4 的 2 倍

### 7.2 API 可用性延迟
- ChatGPT / Codex 首周上线
- API 延迟上线等待额外安全防护到位

---

## 八、已知不足与弱点

### 8.1 SWE-Bench Pro 落后 Claude
- 在某些真实软件工程任务上 Anthropic 仍有微弱优势

### 8.2 MCP Atlas 落后竞争对手
- 工具调用类基准上不是最优

### 8.3 Humanity's Last Exam 退步
- 在此基准上出现了性能退步(regression)
- 某些极端推理任务反倒不如前代

---

## 九、关键结论

1. GPT-5.5 的核心突破是"能用更少 token 完成更多任务"
2. 编程能力最强的模型,但优势并非绝对碾压
3. 知识工作与操作能力是最大增量
4. 安全风险同步升级,被评为最高风险等级
5. 推理效率优化让 2 倍价格实际只增加约 20% 成本
