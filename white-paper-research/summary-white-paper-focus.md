# 白皮书撰写要点手册
## 目标：产出可被 CNN / BBC 正文引用的 AI 行业白皮书

> **来源**：综合 `white-paper-focus.md`（逐篇引用结构分析）、`white-paper-focus2.md`（媒体偏好总结）、`white-paper-focus3.md`（Tier 分级框架与 Checklist）三份调研文件  
> **用途**：撰写白皮书前的参照手册，聚焦 AI 模型层 × SaaS/个人应用层两类主题  
> **更新日期**：2026-07-12

---

## 一、定位共识：白皮书是什么，不是什么

| 是 | 不是 |
|----|------|
| 独立实证研究报告 | 产品宣传册 |
| 行业问题的实测回答 | 公司能力声明 |
| 可被第三方复现的方法论 | Benchmark 截图集合 |
| 量化数据 + 失败案例并列呈现 | "我们是最好的"式结论 |
| 中立语言，承认局限性 | 客户证言 + 路线图愿景 |

> **核心原则（来自 focus3）**：CNN/BBC 记者引用的是"独立实证研究"，而不是"营销材料"。同一份报告，如果被记者认为是厂商背书，就不会出现在正文中——只会被标注为"公司声称…"。

## 二、来源可信度分级（Tier 框架）

根据 CNN/BBC 实际引用频率，来源从高到低分 6 级：

| Tier | 类型 | 典型机构 | CNN/BBC 引用方式 |
|------|------|---------|----------------|
| **T1** | 独立研究机构 | Stanford AI Index、Pew Research、OECD、Brookings、RAND、Oxford Internet Institute、Reuters Institute | 作为事实证据直接引用，不需要平衡其他来源 |
| **T2** | 同行评议学术期刊 | Nature、Science、JAMA、arXiv（有影响力论文）、NeurIPS、ACL、ICLR | BBC 首选；评估模型能力/安全/社会影响时使用 |
| **T3** | 原始测量机构 | Stanford HELM、LMSYS Chatbot Arena、Artificial Analysis、Epoch AI | 提供可重复测试 + 原始数字 + 历史趋势 |
| **T4** | 政府/监管机构报告 | NIST、EU AI Office、UK AISI、FTC、UK CMA、中国工信部 | CNN 首选（政策议题）；政府书面文件 > 所有其他来源 |
| **T5** | 大规模行业调查 | Harmonic Security、TrendForce、Omdia、Interact Analysis、Software AG 调查 | 需大样本 + 透明抽样 + 公开问卷 + 置信区间 |
| **T6** | 企业内部数据（限定条件） | OpenAI 使用统计、GitHub Copilot 数据、Microsoft 财报、Culture Amp HR 数据 | 仅当数据在其他地方无法获得时引用；仅作事实陈述，不作结论 |

> **注意**：T6（公司自有报告）在 BBC 中通常会被独立来源平衡掉。白皮书若由 AI 公司资助，BBC 大概率会在引用时注明利益关系，削弱其可信度。

## 三、CNN vs BBC 引用风格差异

了解差异才能针对性地设计白皮书证据结构：

| 维度 | CNN | BBC |
|------|-----|-----|
| **首选来源** | 政府官员书面文件 + 大型咨询/投行报告 | 顶级大学命名教授 + 同行评议期刊 |
| **专家引用格式** | 姓名 + 职位 + 机构（政府/公司均可） | 姓名 + 大学/研究机构 + 具体职称（多段引语） |
| **学术论文** | 少用（偏好机构报告代替） | 高频使用，需附 DOI 链接 |
| **政府文件** | 核心证据（直接引用原文） | 补充背景 |
| **公司声明** | 与其他来源并列，相对平衡 | 作为"被调查方立场"，必须有独立来源平衡 |
| **批判立场** | 多声部，给各方声音 | BBC Future 对企业声明持明确批判态度 |
| **来源密度** | 深度调查 10+ 来源交叉验证 | 5-8 个来源，但每个附方法论说明 |
| **匿名来源** | 接受（需说明背景） | 极少使用 |

**实操建议**：
- 想被 CNN 引用 → 核心论点附政府文件 URL + 两家以上咨询机构数据交叉验证
- 想被 BBC 引用 → 核心论点有学术期刊支撑 + 命名大学教授背书 + 方法论透明可验证

## 四、AI 模型层 vs SaaS 层：引用来源差异

### 4.1 AI 模型层白皮书

优先引用：
1. **顶级 AI 研究学者**（LeCun、Fei-Fei Li 级别，必须有机构背书）
2. **顶级大学教授**（Oxford, Edinburgh, MIT, CMU 等）
3. **同行评议期刊**（JAMA, Frontiers in AI, Nature Medicine）
4. **独立政府研究机构**（AISI, Mozilla Foundation, NIST）
5. **arXiv 论文**（需标注是否经同行评议）

白皮书专项要求：
- 若评估模型能力 → 需有**独立测试方法论**，不能只用厂商 benchmark
- 必须有**局限性与风险**章节（BBC 对只讲好处的声明持批判立场）
- 政策相关结论 → 引用政府文件 URL，优于转述

### 4.2 SaaS / 应用层白皮书

优先引用：
1. **知名投行研究报告**（Morgan Stanley, Goldman Sachs, BCG, Bain 级别）
2. **专注数据追踪的市场研究机构**（Omdia, TrendForce, Interact Analysis）
3. **以数据追踪为核心业务的安全/分析公司报告**（Harmonic Security, Ahrefs）
4. **工会/公民组织研究报告**（FDA 工会，IPPR 等）
5. **企业 CEO/VP 命名访谈**（姓名+机构+职称，不接受匿名）

白皮书专项要求：
- 使用量数据 → 来自**第三方追踪机构**，而非 AI 公司自我声明
- 企业案例 → 需有管理层命名证言或可引用的企业公开披露
- 市场规模预测 → 引用投行报告（Morgan Stanley/Goldman Sachs 级别）

## 五、可被引用的白皮书：6 大核心特征

### A. 原始数据（最重要）
- 报告必须测量了**别人没测量过的东西**
- 示例：10,000 条 prompt 测试 / 500 个 AI 产品比较 / 200 万次 benchmark 运行
- 记者不引用摘要，他们引用**一手证据**

### B. 透明方法论
必须说明：
- 数据集来源
- 抽样方法
- 评估流程
- 使用的指标
- 局限性
- 置信度/误差范围

### C. 可复现性
- 开放 GitHub 仓库 / 数据集 / Prompt 集合 / 评估脚本 / 原始 CSV
- 记者和研究者可以独立验证

### D. 量化结果
- 用"73%"，不用"很多"
- 图表 > 形容词
- 每个关键论点必须有精确数字支撑

### E. 公共利益议题
BBC/CNN 偏好回答以下类型问题的报告：
- 哪个 AI 模型在真实任务中表现更好？
- 哪个 AI 幻觉率更高？
- 哪个 AI 编程助手节省了更多时间？
- 哪个产品泄露了私人数据？
- AI Agent 在多步骤任务中的成功率是多少？

### F. 平衡结论
高质量报告必须包含：
- 优势
- 劣势
- **失败案例**（缺失此项会大幅降低可信度）
- 已知局限性
- 置信区间

## 六、白皮书结构模板（最可能被引用的格式）

```
1. Executive Summary          ← 记者最先读，必须含所有核心数字
2. Research Questions         ← 明确说明研究什么问题
3. Methodology                ← 数据集、抽样方法、评估流程
4. Dataset Description        ← 样本量、时间范围、来源
5. Evaluation Metrics         ← 使用什么指标、为什么
6. Experimental Design        ← 如何控制变量
7. Main Results               ← 核心发现，量化呈现
8. Failure Analysis           ← 失败案例与边界情况（不可省略）
9. Statistical Analysis       ← 置信区间、显著性检验
10. Practical Implications    ← 对读者有什么意义
11. Limitations               ← 研究局限，诚实表述
12. Appendix                  ← 原始数据 / Prompt 库 / 脚本 / 版本历史
```

> 这个结构接近学术报告，而非产品手册——这是 BBC/CNN 愿意引用的核心原因。

## 七、专家引用规范（必须遵守）

BBC 和 CNN 对专家引用的格式要求极为严格，所有专家引用必须满足：

```
格式：[姓名]，[机构全称]，[具体职称]
示例：Shannon Vallor，University of Edinburgh，数据与 AI 伦理学教授
示例：Lian Jye Su，Omdia Research，首席分析师（专注 AI 与人形机器人）
```

禁止使用：
- "某位专家表示..."（匿名）
- "业界人士认为..."（无归属）
- 仅公司名不带职称（"来自 Google 的研究员"不够）

## 八、机构可信度四项必须声明

白皮书发布前，以下四项声明必须明确写在文档开头：

| 声明项 | 说明 | 缺失后果 |
|-------|------|---------|
| **机构署名** | 清晰可查的出品机构（非匿名个人） | 记者无法验证来源 |
| **独立性声明** | 说明研究独立于被研究的商业实体 | BBC 会标注利益关系 |
| **利益关系披露** | 资金来源（类似学术论文 COI 声明） | 被 BBC 平衡处理 |
| **方法论透明** | 数据采集方法、样本量、时间段 | 降至 T6 级可信度 |

## 九、发布前检查清单（Checklist）

撰写完成后逐项核对：

**数据与方法**
- [ ] 包含原始数据集（或公开访问链接）
- [ ] 透明的方法论说明（样本量、抽样方式）
- [ ] 可重复的实验设计（含 Prompt 集合或评估脚本）
- [ ] 所有结论有精确数字支撑（无模糊表述）
- [ ] 包含失败案例分析
- [ ] 包含统计分析（误差范围/置信区间）

**引用与归属**
- [ ] 每个数据点有稳定 URL（可被记者直接嵌入超链接）
- [ ] 所有专家引用含姓名+机构+职称
- [ ] 区分同行评议期刊 vs 预印本 vs 内部数据
- [ ] 来自 T1-T4 的来源支撑核心论点

**语言与立场**
- [ ] 使用中立语言（无"最佳"、"领先"等绝对化表述）
- [ ] 同时评估了多个竞品（非单一产品颂歌）
- [ ] 包含局限性章节（诚实说明不确定性）
- [ ] Executive Summary 前 2 页可独立引用

**机构合规**
- [ ] 明确的独立性声明
- [ ] 利益关系/资金来源披露
- [ ] 版本历史记录
- [ ] 附录包含可下载的原始资料

## 十、红线：CNN/BBC 不会引用的内容

以下类型的内容会被媒体直接归类为"营销材料"而非"证据"：

- 含营销语言的产品声明
- 无方法论的厂商排名
- 客户证言（"某 Fortune 500 公司使用后效率提升 40%"）
- 愿景声明和路线图
- Benchmark 截图（无原始数据）
- 无原始数据的结论性声明
- 仅引用本公司数据的研究
